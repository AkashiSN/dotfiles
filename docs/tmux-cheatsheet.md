# tmux チートシート

この設定（`dot_tmux.conf`）のキーバインドと挙動をまとめたリファレンス。

- **prefix キー**: `Ctrl-a`（デフォルトの `Ctrl-b` から変更。以下 `<prefix>` と表記）
- ウィンドウ / ペインのインデックスは **1** から開始
- コピーモードは **vi 風**（`mode-keys vi`）

> 表記: `<prefix>` = Ctrl-a、`S-left` = Shift+左矢印、`C-h` = Ctrl+h。

---

## ペイン分割

| キー | 動作 |
| --- | --- |
| `<prefix> -` | 水平分割（上下に分ける） |
| `<prefix> \` | 垂直分割（左右に分ける） |

---

## ペイン移動

prefix **なし**で直接操作できる。

| キー | 動作 |
| --- | --- |
| `S-left` | 左のペインへ |
| `S-down` | 下のペインへ |
| `S-up` | 上のペインへ |
| `S-right` | 右のペインへ |
| `C-o` | 次のペインへ順次移動 |
| `C-h` / `C-j` / `C-k` / `C-l` | Neovim の split と tmux ペインをシームレス移動（左/下/上/右） |

> `C-h/j/k/l` は **vim-tmux-navigator** 連携。フォーカスが nvim なら nvim の分割移動、
> そうでなければ tmux ペイン移動になる（自動判別）。Neovim 側にも同じキーが設定済み。

---

## その他の操作

| キー | 動作 |
| --- | --- |
| `<prefix> e` | 全ペインへの同時入力（synchronize-panes）をトグル。状態を画面表示 |

---

## マウス・コピーモード

- **マウス操作が有効**（`mouse on`）: クリックでペイン選択、ドラッグでリサイズ、ホイールでスクロール
- ホイール上回しでコピーモードに入る
- コピーモードは **vi キーバインド**:

| キー（コピーモード中） | 動作 |
| --- | --- |
| `v` | 選択開始（begin-selection） |
| `y` | 選択をコピーしてコピーモード終了（`set-clipboard on` によりシステムクリップボードにも反映） |
| `C-h` / `C-j` / `C-k` / `C-l` | ペイン移動（左/下/上/右） |

> マウスのドラッグでも範囲選択でき、ボタンを離すとコピーされる。`y` 等は tmux の vi コピーモード標準キー。

---

## Neovim / 端末連携（設定値）

ステータスバーのキーではなく、内部的な連携設定。

| 設定 | 内容 |
| --- | --- |
| truecolor | 外側端末が対応していれば RGB 表示（catppuccin 用） |
| `escape-time 10` | nvim の `<Esc>` を即応させる。0 だと SSH 越しに OSC 応答(背景色問い合わせ等)が分断され先頭に `11;?` 等が漏れるため非ゼロにする |
| `focus-events on` | nvim にフォーカスイベントを伝える（autoread 等） |
| `set-clipboard on` | SSH 越しの OSC52 クリップボードを通過させる |

- ウィンドウ一覧は中央揃え
