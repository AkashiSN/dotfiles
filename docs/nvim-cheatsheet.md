# Neovim チートシート

この設定（`dot_config/nvim/`）のキーバインド・コマンドをまとめたリファレンス。
VSCode ライクな操作感を狙った Leader ベースの構成。

- **Leader キー**: `Space`（スペース。`<leader>` と表記）
- ソース: `init.lua` / `lua/config/*` / `lua/plugins/*`

> 表記について: `<leader>` = スペース、`<S-l>` = Shift+l、`<C-h>` = Ctrl+h、`<cr>` = Enter。
> モードは n=ノーマル, i=挿入, v/x=ビジュアル, o=オペレータ待ち, t=ターミナル。

---

## 起動方法

| コマンド | 動作 |
| --- | --- |
| `vi` / `vim` | `nvim` のエイリアス（通常起動） |

> かつての `ide`（`NVIM_IDE=1 nvim` で codex/claude/terminal の 3 パネルを展開する IDE モード）は
> 撤去した。エージェントの多重化は [herdr](herdr-cheatsheet.md) に一本化し、codex/claude は
> herdr ペインでターミナルから直接起動する。

## 基本操作

| キー | モード | 動作 |
| --- | --- | --- |
| `<leader>w` | n | 保存（`:w`） |
| `<leader>q` | n | 終了（`:q`） |
| `<Esc>` | n | 検索ハイライト解除（`:nohlsearch`） |
| `<` / `>` | v | インデント（選択を保持したまま連続実行可） |
| ダブルクリック / `<cr>` | n | アクティブなペインで自動的に挿入モードへ（後述） |

> `<leader>q` は素の `:q`（ウィンドウ/エディタを閉じる）。**バッファだけ閉じてエディタは残したい**
> ときは `<leader>bd`（バッファ削除）。最後の実ファイルを閉じると `[No Name]` ではなく
> ダッシュボードに戻る（後述）。
>
> **Neovim 自体を終了する**には `:qa`（保存し忘れがあれば拒否）／ `:qa!`（破棄して強制終了）／
> `:wqa`（全保存して終了）を使う。

### 移動・ジャンプ（Vim 標準）

この設定独自ではなく Neovim 標準のキー。`gd` / `gr` 等でジャンプした後に使う。

| キー | 動作 |
| --- | --- |
| `<C-o>` | 直前のジャンプ位置へ戻る |
| `<C-i>` | 戻ったジャンプを進む |

### ダブルクリック / Enter で挿入モード（click-to-insert）

VSCode ライクなモードレス感覚のための仕掛け。ウィンドウを**ダブルクリック**するか、
アクティブなペインで **`<cr>`（Enter）** を押すと、そのバッファ種別に応じて自動で
入力可能になる:

- 通常の編集バッファ → 挿入モード
- ターミナル → ターミナルジョブモード
- neo-tree / ダッシュボード等 → 何もしない（ツリー操作を維持）

**シングルクリック**はフォーカス移動とカーソル位置決めだけ（挿入モードに入らない）。
そのため、ターミナルのログをシングルクリック + ドラッグでビジュアル選択し `y` で
コピーできる。無効化するには `vim.g.click_to_insert = false`。

`<cr>` は入力可能なバッファでのみ挿入モードに入り、それ以外（quickfix で項目へ
ジャンプ等）では既定の `<cr>` 動作に委譲する。neo-tree など独自に `<cr>` を持つ
バッファはそちら（ファイルを開く等）が優先される。

## ファイルツリー（neo-tree）

| キー | 動作 |
| --- | --- |
| `<leader>e` | エクスプローラの開閉（トグル） |
| `<leader>E` | 現在のファイルをツリー上で表示（reveal） |
| `<leader>gg` | Git ステータスパネル（VSCode のソース管理相当） |

ツリー上部のタブ（Files / Buffers / Git）はマウスクリックで表示ソースを切り替え可能。
ドットファイル・gitignore 対象も表示する設定。

**Git ステータスの自動更新:** ツリーが表示されている間は **5 秒ごと**に Git ステータス
（変更マーカーの色）を再計算する。neo-tree は `.git` 監視と libuv ファイル監視で変更を
検知するが、macOS のフォルダ監視は「ファイル内容だけ外部で書き換わった」変更を取りこぼ
しやすく、`git commit` / `checkout` などでも検知が漏れることがあるため、色が古いまま残る。
これを定期ポーリングで追従させている（neo-tree 内部と同じ `GIT_EVENT` を発火するので
Files / Buffers / Git の全ソースが対象。毎回 `git status` を再実行するので内容変更も拾う）。
ツリーが画面に出ていないときは実行しない。即時更新したいときはツリー上で `R`（refresh）。

**Git タブ内:**

| キー | 動作 |
| --- | --- |
| `<cr>` / ダブルクリック | 全変更ファイルを diffview の専用タブで開き、選択ファイルにフォーカス（左=HEAD / 右=作業ツリー。diffview のファイルパネルと Git タブの一覧が一致） |
| `o` | 差分にせず通常どおりファイルを開く |
| `ga` | カーソル下のファイルを stage（`git add`） |
| `A` | 全ファイルを stage（`git add -A`） |
| `gu` | カーソル下のファイルを unstage |
| `gr` | カーソル下のファイルの変更を破棄（revert） |
| `gc` | コミット |
| `gp` | push |
| `gg` | コミットして push |

> 差分を見ずに一覧上で直接ステージしたいとき向け（neo-tree 標準キー）。差分を見て
> から stage したいときは `<cr>` で diffview を開き、パネル内の `-`（[Git](#git) 参照）を使う。

**ヘルプ（日本語ミニヘルプ）:**

| キー | 動作 |
| --- | --- |
| `?` | **日本語ミニヘルプ**（よく使うキーだけをフロート表示。Files / Buffers / Git の表示中ソースで内容を出し分け） |
| `g?` | 標準ヘルプ（そのソースの全キー・英語） |

> neo-tree 標準の `?`（`show_help`）は英語・全キー羅列で目的のキーを探しにくいため、
> diffview と同じく日本語の要点フロートに差し替えてある。`?` を押すと**いま開いている
> ソース**（Files / Buffers / Git）で実際に効くキーだけが日本語で出る（`?`・`q`・`<esc>`
> で閉じる）。英語の全キー一覧が見たいときは `g?`。

| キー | 動作 |
| --- | --- |
| `<S-l>` | 次のバッファ |
| `<S-h>` | 前のバッファ |

> ごと消え、畳み続けると nvim ごと落ちるため、ウィンドウを保つ削除に統一している。

## ターミナル（snacks）

| キー | モード | 動作 |
| --- | --- | --- |
| `<leader>t` | n | ターミナルの開閉（トグル） |
| `<C-/>` | n, t | ターミナルの開閉（トグル） |

**端末内の URL:** `:terminal` バッファ（herdr で動かす claude/codex 等の出力を含む）では、
`http(s)://` が下線ハイライトされ、ノーマルモードで URL 上に `gx` を押すと開く
（`lua/config/terminal-urls.lua`。`TermOpen` で全端末に付与）。カーソル下に URL が無ければ
標準の `gx` にフォールバックする。開く先は `vim.ui.open`＝`$BROWSER` があればそれを優先し、
SSH 先では `$BROWSER=portfwd-open` 経由でローカル mac のブラウザが開く（[portfwd](portfwd-cheatsheet.md)）。

## Git

### 行マーク・hunk 操作（gitsigns）

| キー | 動作 |
| --- | --- |
| `]h` | 次の変更（hunk）へ |
| `[h` | 前の変更（hunk）へ |
| `<leader>gs` | hunk をステージ |
| `<leader>gr` | hunk をリセット |
| `<leader>gp` | hunk をプレビュー |
| `<leader>gb` | 行の blame を表示 |

### 差分・履歴（diffview）

| キー | 動作 |
| --- | --- |
| `<leader>gd` | 作業ツリー vs HEAD の差分を開く |
| `<leader>gq` | diffview を閉じる |
| `<leader>gh` | 現在ファイルの変更履歴 |
| `<leader>gH` | リポジトリ全体の変更履歴 |

**diffview のパネル内:**

| キー | 動作 |
| --- | --- |
| `-` / `s` | カーソル下のファイル / hunk を stage / unstage |
| `S` | 全て stage |
| `U` | 全て unstage |
| `X` | 変更を破棄 |
| `R` | 全ファイル refresh |
| `<cr>` / `o` | カーソル下の差分を開く |
| `<tab>` / `<s-tab>` | 次 / 前の変更ファイルの差分 |
| `?` | **日本語ミニヘルプ**（よく使うキーだけをフロート表示。ファイルパネル / 履歴パネル）|
| `g?` | 標準ヘルプ（そのペインの全キー・英語） |
| `q` | diffview ごと閉じて元のレイアウトに戻る |

> diffview 内では `q` を専用に潰して、diff/パネルどこからでも確実に diffview ごと
> 閉じられるようにしてある。
>
> `g?` の標準ヘルプは**カーソルのあるペイン専用**で英語・項目が多い（stage 系は
> ファイルパネルにフォーカスしないと出ない）。よく使うキーだけ見たいときは
> ファイルパネル/履歴パネルで `?` を押すと日本語の要点だけがフロート表示される
> （`?`・`q`・`<esc>` で閉じる）。差分ウィンドウ側の `?` は素の後方検索のまま。
>
> パネル左上の `Help: ?` というヒントは、この日本語ミニヘルプ（`?`）を指すよう
> 差し替えてある（標準では `Help: g?`）。英語の全キー一覧が見たいときは従来どおり
> `g?` で開ける。

**差分の見た目（`options.lua`）:**

- `fillchars` の `diff` を空白にしてあるため、片側にしか行が無い「埋め草行」が
  ハイフン／スラッシュの壁ではなく、`DiffDelete`（catppuccin の赤系）背景だけで
  表示される。
- `diffopt` に `linematch:60` を設定し、変更行どうしを賢く対応付け。
  行単位で違うときは行内の差分だけが色付けされて見やすい。

## ダッシュボード（最後のファイルを閉じたら表示）

名前付きの実ファイルバッファを全て閉じると、`[No Name]` ではなく Snacks の
ダッシュボードを表示する（`lua/config/autocmds.lua` の `BufDelete` フック）。
閉じ方（タブの ×・`:bd`・`<leader>bd`・`Snacks.bufdelete`）を問わず効き、
残った無名空バッファは掃除する。

## グローバル設定（抜粋）

| 設定 | 値 / 動作 |
| --- | --- |
| Leader / LocalLeader | `Space` |
| マウス | 全モードで有効（`mouse=a`） |
| クリップボード | システム連携（`unnamedplus`）。OS ツール（`pbcopy`/`wl-copy`/`xclip`/`xsel`）があればそれを使い、無ければ OSC52 で端末経由転送（SSH・コンテナ対応） |
| インデント | 既定 tab=4・スペース展開。`lua`/`yaml`/`json`/`ts`/`js`/`tsx`/`terraform`/`hcl` は 2 |
| 検索 | `ignorecase` + `smartcase`（小文字なら大小無視、大文字混在で区別） |
| 分割 | 右・下に開く（`splitright` / `splitbelow`） |
| その他 | undo 永続化（`undofile`）・swap 無効・yank 時ハイライト・前回カーソル位置を復元 |

## 外部ファイル変更の自動同期

`lua/config/autoreload.lua` が外部変更を監視する（`autoread` + フォーカス/無操作時の `:checktime`）。

- **未編集バッファ**: 外部で変更されると通知なしで静かに自動リロードされる。
- **編集中バッファ**: エディタの変更は保持される。保存（`:w`）時に外部変更があると vim 標準の警告（`E13`）が出るので、`:w!` で上書きするか内容を確認する。

## キーマップグループ（which-key）

`<leader>` を押すと候補が表示される。主なグループ:

| プレフィックス | グループ |
| --- | --- |
| `<leader>f` | find（検索） |
| `<leader>b` | buffer（バッファ） |
| `<leader>c` | code（コード） |
| `<leader>g` | git |
| `<leader>t` | terminal（ターミナル） |
