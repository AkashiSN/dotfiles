# Neovim チートシート

この設定（`dot_config/nvim/`）のキーバインド・コマンド・IDE モードの動作をまとめたリファレンス。
VSCode ライクな操作感を狙った Leader ベースの構成。

- **Leader キー**: `Space`（スペース。`<leader>` と表記）
- ソース: `init.lua` / `lua/config/*` / `lua/plugins/*`

> 表記について: `<leader>` = スペース、`<S-l>` = Shift+l、`<C-h>` = Ctrl+h、`<cr>` = Enter。
> モードは n=ノーマル, i=挿入, v/x=ビジュアル, o=オペレータ待ち, t=ターミナル。

---

## 起動方法

| コマンド | 動作 |
| --- | --- |
| `vi` / `vim` | `nvim` のエイリアス（通常起動） |
| `ide [path...]` | **IDE モード**で起動（`NVIM_IDE=1 nvim`）。詳細は[IDE モード](#ide-モード)参照 |

`ide` は zsh 関数（`dot_zshrc`）。VSCode の `code` コマンドと同じ感覚で使える:

| 引数 | 動作 |
| --- | --- |
| `ide` | フルレイアウト（左上＝codex） |
| `ide .` / `ide ~/project` | そのディレクトリへ `cd` してフルレイアウト（左上＝codex） |
| `ide file.go` | ファイルを左上に開いた状態でレイアウト展開（codex はタブで待機） |

## 基本操作

| キー | モード | 動作 |
| --- | --- | --- |
| `<leader>w` | n | 保存（`:w`） |
| `<leader>q` | n | 終了（`:q`） |
| `<Esc>` | n | 検索ハイライト解除（`:nohlsearch`） |
| `<` / `>` | v | インデント（選択を保持したまま連続実行可） |
| ダブルクリック / `<cr>` | n | アクティブなペインで自動的に挿入モードへ（後述） |

> `<leader>q` は素の `:q`（ウィンドウ/エディタを閉じる）。**バッファだけ閉じてエディタは残したい**
> ときは `<leader>bd`（バッファ削除）または `:SmartQ`。IDE モードでは `:q` / `:wq` が
> 自動的に `:SmartQ` / `:SmartWQ` に展開される（通常起動では展開されない）。
>
> **Neovim 自体を終了する**には `:qa`（保存し忘れがあれば拒否）／ `:qa!`（破棄して強制終了）／
> `:wqa`（全保存して終了）を使う。これらは IDE モードでも展開されずそのまま終了するので、
> ダッシュボードを経由する必要はない。

### 移動・ジャンプ（Vim 標準）

この設定独自ではなく Neovim 標準のキー。`gd` / `gr` 等でジャンプした後に使う。

| キー | 動作 |
| --- | --- |
| `<C-o>` | 直前のジャンプ位置へ戻る |
| `<C-i>` | 戻ったジャンプを進む |

### ダブルクリック / Enter で挿入モード（click-to-insert）

VSCode ライクなモードレス感覚のための仕掛け。ウィンドウを**ダブルクリック**するか、
アクティブなペインで **`<cr>`（Enter）** を押すと、そのバッファ種別に応じて自動で
入力可能になる:

- 通常の編集バッファ → 挿入モード
- ターミナル（claude / 下部）→ ターミナルジョブモード
- neo-tree / ダッシュボード等 → 何もしない（ツリー操作を維持）

**シングルクリック**はフォーカス移動とカーソル位置決めだけ（挿入モードに入らない）。
そのため、ターミナルのログをシングルクリック + ドラッグでビジュアル選択し `y` で
コピーできる。無効化するには `vim.g.click_to_insert = false`。

`<cr>` は入力可能なバッファでのみ挿入モードに入り、それ以外（quickfix で項目へ
ジャンプ等）では既定の `<cr>` 動作に委譲する。neo-tree など独自に `<cr>` を持つ
バッファはそちら（ファイルを開く等）が優先される。

## ファイルツリー（neo-tree）

| キー | 動作 |
| --- | --- |
| `<leader>e` | エクスプローラの開閉（トグル） |
| `<leader>E` | 現在のファイルをツリー上で表示（reveal） |
| `<leader>gg` | Git ステータスパネル（VSCode のソース管理相当） |

ツリー上部のタブ（Files / Buffers / Git）はマウスクリックで表示ソースを切り替え可能。
ドットファイル・gitignore 対象も表示する設定。

**Git ステータスの自動更新:** ツリーが表示されている間は **5 秒ごと**に Git ステータス
（変更マーカーの色）を再計算する。neo-tree は本来イベント駆動で、外部コマンド
（`git commit` / `pull` / `checkout` など、作業ツリーのファイルを書き換えない操作）では
色が古いまま残るため、定期ポーリングで追従させている。表示中のソースだけ更新し、
ツリーを閉じている間は実行しない。即時更新したいときはツリー上で `R`（refresh）。

**Git タブ内:**

| キー | 動作 |
| --- | --- |
| `<cr>` / ダブルクリック | 全変更ファイルを diffview の専用タブで開き、選択ファイルにフォーカス（左=HEAD / 右=作業ツリー。diffview のファイルパネルと Git タブの一覧が一致） |
| `o` | 差分にせず通常どおりファイルを開く |
| `ga` | カーソル下のファイルを stage（`git add`） |
| `A` | 全ファイルを stage（`git add -A`） |
| `gu` | カーソル下のファイルを unstage |
| `gr` | カーソル下のファイルの変更を破棄（revert） |
| `gc` | コミット |
| `gp` | push |
| `gg` | コミットして push |

> 差分を見ずに一覧上で直接ステージしたいとき向け（neo-tree 標準キー）。差分を見て
> から stage したいときは `<cr>` で diffview を開き、パネル内の `-`（[Git](#git) 参照）を使う。

**ヘルプ（日本語ミニヘルプ）:**

| キー | 動作 |
| --- | --- |
| `?` | **日本語ミニヘルプ**（よく使うキーだけをフロート表示。Files / Buffers / Git の表示中ソースで内容を出し分け） |
| `g?` | 標準ヘルプ（そのソースの全キー・英語） |

> neo-tree 標準の `?`（`show_help`）は英語・全キー羅列で目的のキーを探しにくいため、
> diffview と同じく日本語の要点フロートに差し替えてある。`?` を押すと**いま開いている
> ソース**（Files / Buffers / Git）で実際に効くキーだけが日本語で出る（`?`・`q`・`<esc>`
> で閉じる）。英語の全キー一覧が見たいときは `g?`。

## Git

### 行マーク・hunk 操作（gitsigns）

| キー | 動作 |
| --- | --- |
| `]h` | 次の変更（hunk）へ |
| `[h` | 前の変更（hunk）へ |
| `<leader>gs` | hunk をステージ |
| `<leader>gr` | hunk をリセット |
| `<leader>gp` | hunk をプレビュー |
| `<leader>gb` | 行の blame を表示 |

### 差分・履歴（diffview）

| キー | 動作 |
| --- | --- |
| `<leader>gd` | 作業ツリー vs HEAD の差分を開く |
| `<leader>gq` | diffview を閉じる |
| `<leader>gh` | 現在ファイルの変更履歴 |
| `<leader>gH` | リポジトリ全体の変更履歴 |

**diffview のパネル内:**

| キー | 動作 |
| --- | --- |
| `-` / `s` | カーソル下のファイル / hunk を stage / unstage |
| `S` | 全て stage |
| `U` | 全て unstage |
| `X` | 変更を破棄 |
| `R` | 全ファイル refresh |
| `<cr>` / `o` | カーソル下の差分を開く |
| `<tab>` / `<s-tab>` | 次 / 前の変更ファイルの差分 |
| `?` | **日本語ミニヘルプ**（よく使うキーだけをフロート表示。ファイルパネル / 履歴パネル）|
| `g?` | 標準ヘルプ（そのペインの全キー・英語） |
| `q` | diffview ごと閉じて元のレイアウトに戻る |

> IDE モードでは `:q` が `SmartQ` に化けて片ペインだけ消える事故を防ぐため、
> diffview 内では `q` を専用に潰してある。
>
> `g?` の標準ヘルプは**カーソルのあるペイン専用**で英語・項目が多い（stage 系は
> ファイルパネルにフォーカスしないと出ない）。よく使うキーだけ見たいときは
> ファイルパネル/履歴パネルで `?` を押すと日本語の要点だけがフロート表示される
> （`?`・`q`・`<esc>` で閉じる）。差分ウィンドウ側の `?` は素の後方検索のまま。
>
> パネル左上の `Help: ?` というヒントは、この日本語ミニヘルプ（`?`）を指すよう
> 差し替えてある（標準では `Help: g?`）。英語の全キー一覧が見たいときは従来どおり
> `g?` で開ける。

**差分の見た目（`options.lua`）:**

- `fillchars` の `diff` を空白にしてあるため、片側にしか行が無い「埋め草行」が
  ハイフン／スラッシュの壁ではなく、`DiffDelete`（catppuccin の赤系）背景だけで
  表示される。
- `diffopt` に `linematch:60` を設定し、変更行どうしを賢く対応付け。
  行単位で違うときは行内の差分だけが色付けされて見やすい。

## IDE モード

`ide`（= `NVIM_IDE=1 nvim`）で起動すると、VSCode ライクなレイアウトを自動展開する。

### レイアウト

```
┌──────────┬───────────────────────┬───────────────────────┐
│          │                       │                       │
│ neo-tree │   codex ペイン         │   claude ペイン        │
│  (左)    │   (左上)               │   (右上)               │
│  全高    │   zsh -ic codex        │   zsh -ic claude       │
│          │   ※ファイル引数時は    │                        │
│          │     ファイルを表示      │                        │
│          ├───────────────────────┴───────────────────────┤
│          │        下部ターミナル(~28%・右領域の全幅)        │
└──────────┴───────────────────────────────────────────────┘
```

- 右領域は上段（codex/エディタ / claude）と下段（ターミナル）の2段。下部ターミナルは
  **左ツリーを除く右領域の全幅**（左上＋claude の真下）に渡る
- 左上（codex/エディタ）と claude ペインは左右 **50/50**（ツリー幅を差し引いて算出）
- **codex（左上）/ claude（右上）の 2 エージェント**を [agmsg](agmsg-cheatsheet.md) 経由で
  会話・相互レビューさせる運用を想定。両者は cwd 直下で起動するので同一チームに参加できる
- ツリーの開閉（`<leader>e` 等）に追従して自動でリバランス。手動なら `:IdeRebalance`
- codex / claude ペインはインタラクティブ zsh 経由で起動（PATH・fnm/node 初期化を読み込むため）
- ターミナル（codex / claude / 下部）はグローバルの `nowrap` を上書きして **折り返し有効**。
  横スクロールせず、スクロールバックも幅に合わせて折り返す

### 端末タブ

- codex / claude / terminal は **buflisted な端末バッファ**として生成され、
  ファイルと同じく **bufferline のタブに `codex` / `claude` / `terminal` と並ぶ**
  （bufferline の `name_formatter` がバッファ変数 `term_label` を表示名に使う）
- **ペインにフォーカスして上部タブをクリックすると、そのペインにそのバッファ
  （ファイル / codex / claude / terminal）が表示される**（自由入れ替え）。
  端末タブを置けば行番号なし＋挿入モード、ファイルタブを置けば行番号付きに自動で切り替わる。
  元のバッファはタブとして残るので、別ペインで再度クリックすれば呼び戻せる
  （= 中身を「置く」セマンティクス。neo-tree／フロート窓へのクリックは無視）
- キーでも同じことができる（下記。`:IdeShow {codex|claude|terminal}` のショートカット）
- **ファイルツリー（neo-tree）からファイルを選ぶと、フォーカス中のペインに開く**。
  端末ペイン（codex/claude/terminal）を選択中でもそのペインにファイルが入る
  （neo-tree 既定の「端末窓は置換しない」を IDE モードでは無効化し、開いた直後に
  行番号付きスタイルへ自動で戻す）

| キー | 動作 |
| --- | --- |
| `<leader>ic` | フォーカス中のペインに **codex** を表示 |
| `<leader>ia` | フォーカス中のペインに **claude** を表示 |
| `<leader>it` | フォーカス中のペインに **terminal** を表示 |
| `<leader>ir` | ペインを左右 50/50 に取り直す（`:IdeRebalance`） |

> これらのキーマップ・タブクリックの入れ替えは **IDE モードのときだけ** 有効
> （`<leader>i` = `ide` グループ）。通常起動ではタブクリックは従来どおりカレント窓の
> バッファ切替（`:buffer`）。

### IDE モード時の挙動

- `:q` / `:wq` は **自動的に `:SmartQ` / `:SmartWQ`** に展開される
  （バッファだけ閉じ、最後の1枚を閉じてもダッシュボードに戻る）
- **Neovim を終了する**なら `:qa` / `:qa!` / `:wqa`。これらは展開されず直接終了する
  （`:q` では終了しないのはこのため）
- ファイルタブ（bufferline）を全て閉じて実ファイルが無くなると、
  `[No Name]` ではなくダッシュボードを表示
- `NVIM_IDE` 環境変数は起動直後に破棄され、子プロセス（ターミナル等）へは伝播しない

## グローバル設定（抜粋）

| 設定 | 値 / 動作 |
| --- | --- |
| Leader / LocalLeader | `Space` |
| マウス | 全モードで有効（`mouse=a`） |
| クリップボード | システム連携（`unnamedplus`）。SSH 越しは OSC52 で転送 |
| インデント | 既定 tab=4・スペース展開。`lua`/`yaml`/`json`/`ts`/`js`/`tsx`/`terraform`/`hcl` は 2 |
| 検索 | `ignorecase` + `smartcase`（小文字なら大小無視、大文字混在で区別） |
| 分割 | 右・下に開く（`splitright` / `splitbelow`） |
| その他 | undo 永続化（`undofile`）・swap 無効・yank 時ハイライト・前回カーソル位置を復元 |

## 外部ファイル変更の自動同期

`lua/config/autoreload.lua` が外部変更を監視する（`autoread` + フォーカス/無操作時の `:checktime`）。

- **未編集バッファ**: 外部で変更されると通知なしで静かに自動リロードされる。
- **編集中バッファ**: エディタの変更は保持される。保存（`:w`）時に外部変更があると vim 標準の警告（`E13`）が出るので、`:w!` で上書きするか内容を確認する。
